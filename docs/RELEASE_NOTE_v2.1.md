# Release Note - Antigravity Proxy Launcher v2.1

## 版本更新

### 功能增强与体验优化

#### 1. 改进运行环境与日志清理机制
**问题描述**: 之前用户在界面的“清理环境”与“清空日志”操作存在语义不明确的问题，容易混淆是否会连带清理日志和系统回退排查文件。

**改进详情**:
- **运行环境清理**: 明确定义“清理环境”不包含日志删除行为。
- **运行日志清理**: 将按钮原“清空日志”改为“清理日志”，其行为现已扩充。除清理应用状态外，新增同时清理以下所有日志：
  - 导出/生成的修复日志 (`patch.log`)
  - 运行时日志目录 (`runtime_logs/` 及其下内容)
  - `/tmp` 目录下的回退及临时日志 (`antigravity_proxy*.log`)
- 并在 UI 实时日志面板中回显已清理的具体目标文件项。

#### 2. 界面与文案优化
- **统一专业话术**: 将部分界面操作按钮中的口语化文案“修复即可”统一修订为标准的“修复应用”，提升专业度与明晰度。

#### 3. 开发环境适配与内置 OAuth Fallback
- 改进 DEBUG 模式下的应用引导：在本地 Xcode 运行时如缺失 Bundle ID，会自动跳过强制退出逻辑并保障开发者可以正常启动 GUI 进行调试。
- 引入了 Team-Shared OAuth Client 支持策略：
  - 支持多级凭证解析机制（环境变量 -> 用户配置 -> 内嵌资源 -> 团队内置回退）。
  - 若用户未进行任何配置，允许回落使用内置凭证，以便全团队开箱即用获取监控配额诊断。
  - 在未配置凭证或鉴权失败时，面板提供“打开 OAuth 配置”操作以明确引导开发者。

#### 4. CLI 修复命令升级
**问题描述**: 当以 CLI 模式编译和构建环境时，输出的帮助命令不够友好。

**改进详情**:
- 将旧的命令输出改为真实的可执行文件名称，并在构建成功后主动提供下一步推荐命令，提高 CLI 使用体验。
- 重构 `compile_without_xcode.sh` 构建脚本的输出体验，提高非 Xcode 纯命令行的可维护性。
